== Automatic Mule internal HTTP headers filtering

To avoid Mule internal HTTP headers to be leaked to the backend implementation or in the response, two filters are defined:
```
<custom-processor class="com.mulesoft.gateway.extension.ProxyRequestHeadersProcessor" />
```
```
<custom-processor class="com.mulesoft.gateway.extension.ProxyResponseHeadersProcessor" />
```

There are no customization for them.
